# 块级作用域的出现

因为Js本身的var拥有全局作用域，会导致在一些不能访问到变量的地方访问到变量，也就是变量的生命周期不同于其他的语言。 所以就出现了let和const

# let和const

let和const的出现让Js也拥有了类似于别的语言的块级作用域，也不会存在var的变量提升问题，在变量外层的作用域不再能访问到这个变量。

if(false){
	let value = 1
}
log(value) // value is not defined

# 不污染全局作用域

在原来ES6之前如果在全局作用域中使用var去声明一个变量，这个变量会作为windows的一个属性被添加，但是使用let和const不会有这种问题

var value = 1
console.log(window.value) // 1
lat value1 = 1
console.log(window.value1) // undifined

# 暂时性死区

由于let和const不会被引擎变量提升所以导致了在引擎遇到let和const的时候就会形成一个临时死区，如果在声明之前访问这些变量就会报错。

# 循环中的块级作用域

如果在循环中利用var和let声明变量，循环的结构会有不同的区别let模仿了js的闭包重写了let循环中的声明

# 结论

把var直接忘掉，多数情况使用const除非这个变量需要进行后续操作或者进行变化才使用let进行声明。

# 没有this

箭头函数内部不推荐使用this，this指向最近的一层不是箭头函数的this，也就是会顺着向上找，因为箭头函数没有this，所以不能使用call apply bind 重新制定this指向

# 没有arguments

箭头函数没有自己的args对象，因为箭头函数可以访问外围函数的 arguments 对象

# 不能new出来

箭头函数没有构造函数，所以不能通过new关键字new出来，一般调用new的时候执行的就是对应对象的构造函数返回实例对象

# 箭头函数又被称为匿名函数

所以匿名函数是没有原型对象的，如果你使用prototyoe去调用原型得到的是undefined

# Symbol类型

# 基本用法

ES6引入了一个独特的数据类型也就是Symbol类型，用来表示独一无二的唯一值，Symbol是一种数据类型而不是对象所以不能new出来

var s = Symbol();

# 迭代器与forOf

# 基本用法

为了解决多重循环代码复杂的问题，ES6提供了迭代器和forof的方法共同解决

所谓迭代器，其实就是一个具有 next() 方法的对象，每次调用 next() 都会返回一个结果对象，该结果对象有两个属性，value 表示当前的值，done 表示遍历是否结束。

//es5
function createIterator(item){
    var i = 0;
    return {
        next:function(){
            var done = i >= item.length;
            var value = !done ? items[i++] : undefined;
            return {
                done: done,
                value: value
            };   
		}
    }
}

forof之能对实现了迭代器接口的数据结构，也就是可遍历的数据对象。

const obj = {
	a:1;
    b:2;
}
for(value of obj){
    console.log(value)
}
//obj is not iterable
手动添加迭代器
obj[Symbol.iterator] = function(){
	return createIterator([1,2,3]) 
}

forof在ES6内默认能使用的数据类型： 1.数组 2.Set 3.Map 4.类数组对象 5.Generator(生成器对象) 6.字符串 为了更好的访问对象中的内容，比如有的时候我们仅需要数组中的值，但有的时候不仅需要使用值还需要使用索引，ES6 为数组、Map、Set 集合内建了以下三种迭代器：

  1. entries() 返回一个遍历器对象，用来遍历[键名, 键值]组成的数组。对于数组，键名就是索引值。
  2. keys() 返回一个遍历器对象，用来遍历所有的键名。
  3. values() 返回一个遍历器对象，用来遍历所有的键值。

# Set数据结构

# 基本用法

Set本身是一种数据结构，类数组式存储元素，跟数组不太一样的就是Set中不能出现重复元素

//初始化
let set = new Set()

set = new Set(document.querySelectorAll('div'));
console.log(set.size); // 66

# 数组去重

# 属性和方法

  1. add(value)：添加某个值，返回 Set 结构本身。
  2. delete(value)：删除某个值，返回一个布尔值，表示删除是否成功。
  3. has(value)：返回一个布尔值，表示该值是否为 Set 的成员。
  4. clear()：清除所有成员，无返回值。
  5. keys()：返回键名的遍历器
  6. values()：返回键值的遍历器
  7. entries()：返回键值对的遍历器
  8. forEach()：使用回调函数遍历每个成员，无返回值 注意 keys()、values()、entries() 返回的是遍历器

# Map数据结构

# WeakSet与WeakMap

在本质上差别不大，都是存储不重复的值的合计，但是WeakSet和WeakMap只能存放对象而且对于对象的引用都是弱引用，在这里的弱引用的意思是虽然这个对象真正被weakset引用但是不会影响引擎的垃圾回收机制，就是不考虑这一层引用关系，一旦对象被设置为null可以正常被回收 eg：WeakMap没有遍历方法也就是没有keys等返回迭代器的方法

# Promise

这个就凭个人感觉了，大部分人聊到Promise都会从回调地狱开始聊起来，都会说Promise是一种解决回调地狱的问题，但我感觉，Promise本身只是一种同步代码的异步执行方案，也就是简化了回调函数的书写方法，通过添加一些特性，让异步更加好执行，比如promise只能更改一次状态，也就代表了，一个promise对应的任务只能执行一次，而且通过promise.race promise.all等方法让回调函数的嵌套以及执行更加简单 而真正解决了回调地狱的应该是通过生成器函数和promise一起重写的await async两个关键字

# Promise红绿灯问题

如题，红灯3s亮一次，绿灯1s亮一次，黄灯2秒亮一次，如何让三个灯不断交替闪烁亮灯？

function red(){
    console.log('red');
}
function green(){
    console.log('green');
}
function yellow(){
    console.log('yellow');
}

var light = function(timer,cb){
    return new Promise(function(resolve,reject){
        setTimeOut(function(){
            cb()
            resolve();
        },timer)
    })
}
var step = function() {
    Promise.resolve().then(function(){
        return light(3000, red);
    }).then(function(){
        return light(2000, green);
    }).then(function(){
        return light(1000, yellow);
    }).then(function(){
        step();
    });
}

step();

# defineProperty

Object.defineProperty(obj, prop, descriptor)
// obj:需要定义属性的对象
// prop:要定义或者修改属性的名称
// descript:描述符
var obj = {};
Object.defineProperty(obj, "num", {
    value : 1,
    //一些ES6对于对象属性的控制类似可读可写等等
    //get 与 set也属于描述符中的一种
    writable : true,
    enumerable : true,
    configurable : true
});
//  对象 obj 拥有属性 num，值为 1
var obj = {}, value = null;
Object.defineProperty(obj, "num", {
    //存取器属性
    get: function(){
        console.log('执行了 get 操作')
        return value;
    },
    set: function(newValue) {
        console.log('执行了 set 操作')
        value = newValue;
    }
})
obj.num = 1 // 执行了 set 操作
console.log(obj.num); // 执行了 get 操作 // 1

这么对比下来我们只需要在set操作中进行这个数据相关页面的修改就可以达到一种双向绑定的原理 但是defineProperty也有一定的局限性，比如他无法对你手动添加的属性进行监听，比如obj中手动定义的obj.age他就无法自动添加监听，而且针对数组这种复杂数据类型，通过对原生方法的寄生式重写，也只能监听到原有七个方法

# AMD-RequireJS

AMD是Requirejs在推广过程中对模块定义的规范化产出也就是AMD规范，只要正确书写，require就可以正确加载依赖，从而进行解析模块 AMD本身有一个预加载过程，会将引入的所有依赖正常加载完毕后再开始正常执行原代码，但是如果依赖关系太过于复杂或者文件过去大，会导致渲染堵塞，也就是预加载的过程太长，从而导致页面异常

    console.log(squareModule.square(3))

//2.CMD推崇就近依赖

# CommonJS

与AMD和CMD最大的不同就是，这两者运行于浏览器端，也就是客户端的模块化，但是CommonJS是针对Node也就是前端的服务端的规范化,但是CommonJS采用了和CMD一样的加载方法也就是按需加载只有在执行到require的时候才去加载对应的模块

//到处模块
var add = function(x,y){ 
	return x + y
}

module.export.add = add
//引入模块
var add = require('./add.js')

！！AMD CMD 都是异步加载方案，但是CommonJS是同步加载方案，服务端一般不用考虑异步的加载方案

Vue的模块化使用的就是ES6的模块化模式，所以可能比较熟悉一点

浏览器引入ES6模块有一些不太一样
<script src="vender/main.js" type="module"></script>
需要在这里协商module的类型

CommonJS引入的是一份值的复制但是如果这个值是引用数据类型，对应的修改还是有效的，但是如果是个值的话，会被浏览器缓存后输出，修改是没有用的

ES6 模块的运行机制与 CommonJS 不一样。JS 引擎对脚本静态分析的时候，遇到模块加载命令 import，就会生成一个只读引用。等到脚本真正执行时，再根据这个只读引用，到被加载的那个模块里面去取值。换句话说，ES6 的 import 有点像 Unix 系统的“符号连接”，原始值变了，import 加载的值也会跟着变。因此，ES6 模块是动态引用，并且不会缓存值，模块里面的变量绑定其所在的模块。

# Webpack

Babel会将ES6的模块打包成一种类CommonJS的模块化模式，但是这种模式浏览器并不能直接加载，所以需要借助一个第三方打包工具Webpack

浏览器之所以无法运行CommonJS是因为浏览器本身并没有CommonJS的相关依赖，webpack就注入了这些依赖，也就是把这些环境给准备好后再去运行CommonJS模块化
