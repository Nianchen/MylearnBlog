1正则表达式

1.1横向模糊匹配 横向拓展 比如 /ab{2，5}c/g 这里面的b有2~5个 虽然数量发生了改变但是都是b一个字母 就是横向增长了 但是没有纵向增长

1.2，纵向模糊匹配· 出现在一个位置的元素可以匹配多个字母 不如 /a[123]b/g 中间的位置就可以是 1 2 或者 3

2字符组

2.1范围表示
虽然叫做字符组 但是说的是这一个字符点上可以出现的字符集合
所以就出现了这种范围的写法
eg /[a-z]/ 代表所有小写字符 /[0-9]/所有数字
但是如果只用匹配几个字符要注意不能让他直接转义成范围写法

2.2排除字符串
也就是这个位置除了这几个东西都能出现 也就是范围取反 /[^abc]/除了abc都行

2.3简写模式
\d就是[0-9]。表示是一位数字。记忆方式：其英文是digit（数字）。
\D就是[^0-9]。表示除数字外的任意字符。
\w就是[0-9a-zA-Z_]。表示数字、大小写字母和下划线。记忆方式：w是word的简写，也称单词字符。
\W是[^0-9a-zA-Z_]。非单词字符。
\s是[ \t\v\n\r\f]。表示空白符，包括空格、水平制表符、垂直制表符、换行符、回车符、换页符。记忆方式：s是space character的首字母。
\S是[^ \t\v\n\r\f]。 非空白符。
.就是[^\n\r\u2028\u2029]。通配符，表示几乎任意字符。换行符、回车符、行分隔符和段分隔符除外。记忆方式：想想省略号...中的每个点，都可以理解成占位符，表示任何类似的东西。


3.量词
3.1了范围也就可以重复出现 比如{m,n}出现的范围就是m~n次数 m<n
{m,} 表示至少出现m次。
{m} 等价于{m,m}，表示出现m次。
? 等价于{0,1}，表示出现或者不出现。记忆方式：问号的意思表示，有吗？
+ 等价于{1,}，表示出现至少一次。记忆方式：加号是追加的意思，得先有一个，然后才考虑追加。
* 等价于{0,}，表示出现任意次，有可能不出现。记忆方式：看看天上的星星，可能一颗没有，可能零散有几颗，可能数也数不过来。
3.2 贪婪匹配和惰性匹配
贪婪匹配就是一定要完成你给的正则表达式最好的返回结果 比如 
let regex=/\d{2,5}/g;
let string = "123 1234 12345 123456"
string.match(regex);
得到的结果是 123 1234 12345 12345 就是数字可以出现两到五次 但是既然能满足就一直往下走直到停了

相反惰性匹配就是尽量进行少次数的对比
let regex = /\d{2,5}？/g;
let string = "123 1234 12345 123456"
string.match( regex )
得到的结果是 12 12 34 12 34 12 34 56 他每次只会进行最小次数的对比

//通过在量词后面加个问号就能实现惰性匹配，因此所有惰性匹配情形如下
//{m,n}?
//{m,}?
//??
//+?
//*?


4.多选分支
一个正则匹配中可以实现多个模式 只需满足其中一个模式就可以 而且这个模式匹配也存在惰性如果前面的成功匹配了后面的东西就不会接着进行匹配
var regex = /good|nice/g;
var string = "good idea, nice try.";
console.log( string.match(regex) ); 
// => ["good", "nice"]
var regex = /goodbye|good/g; 
var string = "goodbye";
console.log( string.match(regex) );
// => ["goodbye"]
 

5.简单案例
匹配16进制颜色
eg： #ffbbad #FFF #ffE
分析都是以#开头
var regex = /#([0-9a-fA-F]{6})/g
匹配24小时时间
eg： 23:45 06:35
var regex = /([0-1][0-9]|[2][0-3]):[0-5][0-9]$/
