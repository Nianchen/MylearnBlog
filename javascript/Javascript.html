JS是目前比较流行的一种脚本语言

所谓脚本语言就是 在引擎或者编译器看来，代码一行一行运行，保证上一句代码运行完后才进行下一句。

JS其实包含了JS基础语法，dom，bom三种东西

浏览器内核用于处理H5 CSS语言 而JS引擎专门用来解析js代码

因为js本身也是一种高级编程语言，他也需要引擎将其解码为机器语言后被计算机识别

1.JS的输入输出语句

JS函数

JS的函数 与 C的函数 比较相似，同样需要先声明后调用

但是声明方法有两种 一种是命名声明，后面通过函数名字进行调用函数，但是也可以匿名通过函数表达是进行声明，但是本身赋值给一个变量，也就是一个标量存储的是一个函数

函数的参数关于可以存在多个，js内置了一个对象进行统计参数个数并且可以调用，但是自身是个伪数组，不能进行添加删除等操作。

函数也存在返回值 未定义返回值返回未定义。

JS的作用域 与其他编程语言相差不大，都分为局部变量和全局变量

同样适用变量调用规则。

JS的代码存在一个预编译的操作 会自动将代码里所有的命名和函数提前到所在作用域的最前端 进行编译，所以这个过程会导致匿名函数 也就是变量的调用错误 一定要确定函数调用的位置。

JS中不像Java 存在class创建类后new出对象

JS中 function 是函数声明 但是再js中 对象同样由 function 声明

new 关键字和其他语言差不多 都是先new出一个空对象 然后利用空对象 中的this 进行为这个对象赋值

对象中的属性 也是通过.调用 同时也能利用强for循环进行遍历

JS的对象分为 自定义对象 内置对象 浏览器对象

而这个内置对象 就像java中的 类调用api一样

JS 同样拥有巨大的api库 具体使用还得去w3c 查询

JS中的string 变量的重新赋值 也是 重新申请的内存 然后改变了指向 原有内存并没有释放

所以在实际的开发中 请避免重复大量的拼接字符串。 会导致内存无意义浪费

其实在string对象 以及 Array中 很多常见方法都是重复的。

JS的更具体的数据类型

简单数据类型 存放在栈中如String Number 等等 复杂数据类型 存放在堆中 栈存放地址 如对象 数组

复杂数据类型 相当于C的传参数传地址的情况 如果通过地址对值进行修改 就会导致 原有的数据进行修改 修改的是地址对应的数据

执行上下文

在js的运行机制里 存在一个控制作用域的概念 被称为 执行上下文 最常见的上下文 就是全局执行上下文 也就是 windows对象的执行上下文 同样不同的函数在执行的时候 也有自己 的函数执行上下文 且在运行的过程中 如果在 自身的函数上下文里并没有找到 就会往上去找 也就是反过来去找 windows的上下文 里 有没有自己需要的属性

定义变量

1.在es6之前 只有一种定义变量的关键字 也就是 var 但是在es6之后 let 和const 的出现 让var 用的更少了 同时 var定义的变量 会在首次预编译的时候 实现一种操作叫做 变量提升 以及 函数提升 仅限于 var 出来的变量 但是不会进行 赋值 也就是 在这个过程中 如果后面提前调用了 这个变量 那么报错 报的应该是 undefined 而不是 Error

2.let let与var的区别不是特别大 它主要的意思是在一个作用域中声明的东西 在用完以后会自动进行销毁 减少内存占用

3.const 就只能用于 常量声明了 这个常量是单一类型 不能被修改
